## 🎯 两种使用方式

### 方式 1：打包时自动签名（推荐）✨

**一步到位，打包+签名全自动**

```bash
python build_with_signing.py
```

**流程：**
```
步骤 1: 检查文件和依赖
步骤 2: 清理旧文件
步骤 3: PyInstaller 打包 → 生成 Sap_Hour_Operate_theme.exe
步骤 4: 数字签名 → 使用 code_signer 模块自动签名
步骤 5: 验证签名 → 检查签名是否成功
```

**生成的文件：**
```
dist/Sap_Hour_Operate_theme.exe (已签名)
```

---

### 方式 2：单独对 EXE 签名

**如果您已经有 EXE 文件，只需签名**

```bash
python SAP签名工具.py
```

**流程：**
```
步骤 1: 初始化签名系统（code_signer）
步骤 2: 显示证书信息
步骤 3: 查找 EXE 文件（dist/Sap_Hour_Operate_theme.exe）
步骤 4: 检查文件签名状态
步骤 5: 执行数字签名
步骤 6: 验证签名结果
```

code_signer/
├── __init__.py           # 模块入口

- 提供与旧版本的兼容接口
- 支持 JSON 配置文件

**3. signature_config.json**（配置文件）
```json
{
  "signature": {
    "enabled": true,
    "default_certificate": "sap_hour_operation",
    "certificates": {
      "sap_hour_operation": {
        "sha1": "144ac4069565211ab67d25a9d6d33af0e18e511e"
      }
    }
  }
}
```

## 🔍 验证签名是否成功

```

2. 选择"属性"
3. 点击"数字签名"标签
4. 查看证书详细信息
```

### 方法 2：PowerShell

```powershell
Get-AuthenticodeSignature dist\Sap_Hour_Operate_theme.exe | Format-List
```

**预期输出：**
```
Status        : Valid
SignerCertificate: [证书详细信息]
...
```

### 方法 3：signtool

```bash
signtool verify /pa /v dist\Sap_Hour_Operate_theme.exe
```

## ❓ 常见问题

### Q: 如何确认签名已集成到打包流程？
**A:** 检查 `build_with_signing.py` 中的以下代码：

```python
# 步骤 5: 签名（在第 536 行）
signer, signer_type = initialize_signer()

if signer:
    # 使用签名模块
    sign_success, sign_message = sign_exe_file_with_module(exe_path, signer, signer_type)
else:
    # 使用备用方法
    sign_success, sign_message = sign_exe_file_fallback(exe_path)

# 步骤 6: 验证签名（在第 552 行）
if sign_success:
    verify_signature(exe_path)
```

### Q: 如果签名失败怎么办？
**A:**
1. 检查证书文件是否存在：`170859-code-signing.cer`
2. 检查配置文件是否正确：`signature_config.json`
3. 确认 Windows SDK 已安装（包含 signtool.exe）
4. 使用独立的签名工具：`python SAP签名工具.py`

### Q: 可以使用其他证书吗？
**A:** 可以！修改 `signature_config.json`：

```json
{
  "certificates": {
    "my_certificate": {
      "name": "我的证书",
      "sha1": "您的证书SHA1",
      "subject": "证书使用者",
      "issuer": "证书颁发者"
    }
  }
}
```
